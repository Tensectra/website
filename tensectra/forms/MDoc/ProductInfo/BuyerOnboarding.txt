SECTION 12 Ś HOW TO GET TEMPLATE UPDATES
????????????????????????????????????????????????????????????????????????????????????????????????

  When TensectraKit releases a new version, you receive an email notification.
  Getting the update is two commands:

  STEP 1 Ś Pull the latest template source:
      cd ~/tensectrakit-core      (or wherever you cloned it)
      git pull origin main

  STEP 2 Ś Re-register the updated template:
      dotnet new install . --force

  ? Done. Any NEW service you scaffold from this point uses the updated template.

  ?????????????????????????????????????????????????????????????????????????????????????????????
  IMPORTANT: DOES UPDATING THE TEMPLATE CHANGE MY EXISTING PROJECTS?
  ?????????????????????????????????????????????????????????????????????????????????????????????

  NO. Your existing generated projects (e.g. MyLoanService) are YOURS.
  The template update only affects NEW projects you scaffold AFTER the update.

  This is intentional Ś you own your generated code. TensectraKit never
  auto-modifies your production codebase.

  To apply improvements from a new version to an existing project:

    a. Read CHANGELOG.md in the tensectrakit-core repo.
       Each entry describes exactly what changed and why.

    b. Scaffold a fresh reference project:
           dotnet new tensectrakit -n TKReference ...

    c. Diff TKReference against your existing project to see the changes.
           (Use Visual Studio "Compare Files", VS Code "Compare with...", or git diff)

    d. Manually apply the improvements you want to your project.
       Ignore changes that do not apply to your domain.

  This pattern is used by all scaffold products (Bullet Train, SAAS Pegasus, Shipfast).
  Your code is never held hostage by a template engine.

SECTION 14 Ś YOUR LICENSE KEY
????????????????????????????????????????????????????????????????????????????????????????????????

  Your license key is in your purchase receipt email.
  It follows the format:   TK-2026-XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX

  WHERE TO PUT IT:

  In your generated project, open:
    <YourServiceName>\Presentation\<YourServiceName>.Api\appsettings.json

  Add your key to the SystemSettings section:

      "SystemSettings": {
        "LicenseKey": "TK-2026-XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX",
        "EncryptionKey": "YourSecretKey1234567890123456!!",
        "UseEncryptedData": false,
        "EncryptResponseData": false,
        ...
      }

  WHAT HAPPENS AT STARTUP:

    - Valid key:   Console logs "TensectraKit Ś Licensed to: YourName (your@email.com)"
    - Missing key: Console logs a warning banner (app still runs Ś not blocked)
    - Invalid key: Console logs a warning banner (app still runs Ś not blocked)

  ?  The license key is validated LOCALLY (offline). The app never phones home.

SECTION 16 Ś TROUBLESHOOTING
????????????????????????????????????????????????????????????????????????????????????????????????

  PROBLEM: "dotnet new install ." says "No templates were found"
  FIX:     Make sure you are inside the tensectrakit-core folder (not a subfolder).
           The .template.config/template.json must be reachable from your current directory.
           Run:  ls .template.config   (should show template.json and ide.host.json)

  PROBLEM: "dotnet new tensectrakit" is not recognized after install
  FIX:     Close and reopen your terminal. Template registration requires a new shell session.
           Then verify:  dotnet new list | grep tensectrakit

  PROBLEM: Build errors after scaffolding
  FIX:     Run  dotnet restore  first.
           If errors persist, check that --EncryptionKey is exactly 32 characters.
           Open appsettings.json and verify all TSKT_* placeholders were replaced.
           Any remaining TSKT_* string means a parameter was not passed to dotnet new.

  PROBLEM: "Access denied" when cloning the private repo
  FIX:     Ensure you accepted the GitHub invitation (Section 2).
           Check you are logged in to the correct GitHub account in your terminal.
           Run:  git config user.name  and  git config user.email  to verify.
           If using HTTPS, ensure your GitHub Personal Access Token has repo read scope.

  PROBLEM: EF Core migration fails Ś "No DbContext was found"
  FIX:     The --startup-project must point to the .Api project (it contains appsettings.json).
           The --project must point to the .Infrastructure project (it contains DbContext files).
           Ensure DatabaseProviderName in appsettings.json matches the --context flag you used.

  PROBLEM: App starts but Swagger shows no endpoints
  FIX:     Check that TensectraKit.Api is set as the startup project in Visual Studio.
           In VS Code / terminal, confirm you are running the .Api project, not another.

  PROBLEM: Redis connection refused at startup
  FIX:     Start the Redis container:  docker start tensectra-redis
           Then verify:  docker exec -it tensectra-redis redis-cli ping  ? should print PONG
           Set AbortOnConnectFail=false in appsettings.json for local dev.

  PROBLEM: License key warning banner in logs
  FIX:     Add your license key to appsettings.json under SystemSettings:LicenseKey.
           The key is in your purchase receipt email (format: TK-2026-XXXXX...).
           The warning is cosmetic Ś the app runs fully without a valid key.

SECTION 17 Ś QUICK REFERENCE CARD  (pin this to your screen)

  FIRST-TIME SETUP (one machine, one-time):
      git clone https://github.com/tensectra-pro/tensectrakit-core
      cd tensectrakit-core
      dotnet new install .

  SCAFFOLD A NEW SERVICE:
      dotnet new tensectrakit -n <YourServiceName> \
        --DatabaseProvider PostgreSQL \
        --DbConnectionString "<your-connection-string>" \
        --EncryptionKey "<exactly-32-chars>" \
        --RedisMode Standalone \
        --RedisEndpoint "localhost:6379" \
        --RedisPassword "" \
        --AbortOnConnectFail false \
        --KafkaBootstrap "localhost:9092"

  GET TEMPLATE UPDATES:
      cd ~/tensectrakit-core
      git pull origin main
      dotnet new install . --force

  RUN MIGRATIONS (PostgreSQL):
      dotnet ef migrations add InitialCreate \
        --project <Service>\Infrastructure\<Service>.Infrastructure \
        --startup-project <Service>\Presentation\<Service>.Api
      dotnet ef database update \
        --project <Service>\Infrastructure\<Service>.Infrastructure \
        --startup-project <Service>\Presentation\<Service>.Api

  RUN THE APP:
      cd <Service>\Presentation\<Service>.Api
      dotnet run
